面向数据的传输协议集合:
    协议集合由多个协议体组成
    协议集合则使用固定的结构模板
    协议体的设计取用最小组成单元

协议集合:
    多个协议体组成

协议体:
    规定一个协议功能的组合单元
    协议体由多个编号连续的协议包组成
    使用协议集合提供的固定结构模板
    添加内部实现细节完备生成

协议包:
    单次协议传输与处理的最小单元
    协议包都有一个严格对应的协议应答
    协议包的数据长度可以不固定
    协议包是按照包编号传输
    传输时可根据实际情况混杂传输
    
扩展理解:
    所有的协议包产生即相当于生成了协议包事件
    无论协议包由外界触发生成还是内部触发生成
        (外界触发生成:rx实际的协议包需要处理)
        (内部触发生成:rx虚假的协议包需要主动tx真实的协议包)
    意味系统接收到该协议包事件
    此时协议模组响应该协议包事件并处理
    那么对于协议包的rx与tx事件整体流程即变得清晰明了
    无论是发送者还是接收者都可以使用一个严格固定的流程
    每一次都一定是rx用来触发tx而无论这个rx是虚假的还是真实的
    发送者视角:
        rx到一个需要tx的协议包(fake ack)
        tx协议包(ack + data)...
        超时等待...
        rx协议包(ack)
    接收者视角:
        rx到一个需要解析的协议包(ack + data)
        解析处理...
        tx协议包(ack)
    这个语义可以让通信双方形成一个严格自锁的逻辑闭环
    每一份事件都一定有一个完整的响应链路
    因为发送的语义被转化成了接收的语义
    发送本身的语义被转化成了生成发送的事件
